# Role-Based UX Design

캠페인 스태프 역할별 워크플로우 기반 UI 설계

---

## Staff Roles & Daily Workflows

### 1. 총무팀 (General Affairs)

**역할:** 행정, 물류, 일정 조율, 예산 관리

**Typical Day:**
```
08:00  출근, 오늘 일정 확인
08:30  현장 행사 준비물 체크리스트 확인
09:00  자원봉사자 배치 조율
10:00  경비 지출 승인 요청 처리
12:00  오후 행사 장소 최종 확인
14:00  긴급 요청 대응 (타팀 지원)
16:00  내일 일정 사전 준비
18:00  일일 정산 및 보고
```

**Pain Points:**
- 여러 화면 이동해야 오늘 할 일 파악
- 승인 대기 건이 어디 있는지 찾기 어려움
- 긴급 요청 놓치기 쉬움

**Top 3 Actions (3초 내 접근 필요):**
1. ⏰ **오늘의 일정** - 시간순 행사/미팅 목록
2. ✅ **승인 대기** - 결재 필요한 건 (경비, 물품)
3. 🔥 **긴급 요청** - 나에게 할당된 급한 작업

---

### 2. 정책팀 (Policy)

**역할:** 정책 연구, 공약 개발, 팩트체크, 토론 준비

**Typical Day:**
```
08:00  뉴스 클리핑 검토
09:00  상대 후보 발언 팩트체크
10:00  정책 브리프 작성
12:00  후보자 질의응답 자료 준비
14:00  공약 세부사항 연구
16:00  토론 예상 질문 대응 자료
18:00  내일 뉴스 브리핑 준비
```

**Pain Points:**
- 마감 임박 문서를 놓침
- 팩트체크 요청이 여러 채널로 들어옴
- 정책 문서 버전 관리 혼란

**Top 3 Actions (3초 내 접근 필요):**
1. 📋 **마감 임박** - D-Day 기준 정렬된 정책 문서
2. 🔍 **팩트체크 큐** - 확인 요청 들어온 건
3. 📰 **오늘의 이슈** - 대응 필요한 뉴스/발언

---

### 3. 홍보팀 (Media/SNS)

**역할:** SNS 운영, 언론 대응, 콘텐츠 제작, 여론 모니터링

**Typical Day:**
```
08:00  밤새 SNS 반응 체크
09:00  오늘 콘텐츠 스케줄 확인
10:00  카드뉴스/영상 제작
12:00  점심 포스팅
14:00  실시간 이슈 대응 콘텐츠
16:00  언론 문의 대응
18:00  저녁 콘텐츠 발행
20:00  engagement 분석
```

**Pain Points:**
- 예약 발행 콘텐츠 상태 확인 어려움
- 실시간 트렌드 대응이 느림
- 승인 대기 중인 콘텐츠 병목

**Top 3 Actions (3초 내 접근 필요):**
1. 📱 **오늘 발행** - 예약된 콘텐츠 타임라인
2. 🔥 **트렌딩 대응** - 급히 대응할 이슈
3. ⏳ **승인 대기** - 발행 전 검토 필요 콘텐츠

## Dashboard Redesign

### Before (기존)

```
┌─────────────────────────────────────────────────────────┐
│  Dashboard                                              │
├─────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ Active   │  │ Upcoming │  │ Pending  │              │
│  │ Tasks    │  │ Events   │  │ Approvals│              │

│  └──────────┘  └──────────┘  └──────────┘              │
│                                                         │
│  Recent Tasks                    Upcoming Events        │
│  ├─ Task 1                       ├─ Event 1            │
│  ├─ Task 2                       ├─ Event 2            │
│  └─ Task 3                       └─ Event 3            │
└─────────────────────────────────────────────────────────┘
```

**문제점:**
- 모든 역할에게 동일한 화면 (One-size-fits-all)
- 역할별 우선순위 반영 안 됨
- "내가 지금 해야 할 것"이 명확하지 않음
- 3초 내 행동 불가 → 클릭 3번 이상 필요

## Key UX Changes Explained

### 1. 역할 인식 개인화 (Role-Aware Personalization)

**변경:** 로그인 시 역할(department) 기반으로 대시보드 섹션 순서 및 내용 변경

**이유:**
- 총무팀에게 팩트체크 큐는 무의미
- 홍보팀에게 예산 소진율은 우선순위 낮음
- 각 역할의 "첫 번째 행동"이 다름

**구현:**
```typescript
const dashboardSections = {
  'general-affairs': ['schedule', 'urgent', 'approvals', 'stats'],
  'policy': ['deadlines', 'factcheck', 'issues', 'documents'],
  'media': ['publishing', 'trending', 'pending-content', 'analytics'],
};
```

---

### 2. "내 작업" 최상단 고정 (My Tasks First)

**변경:** Tasks 페이지 최상단에 "내 작업" 섹션 분리

**이유:**
- 기존: 칸반 보드에서 내 작업 찾으려면 스크롤 + 필터
- 개선: 페이지 열자마자 내 할 일 3개 즉시 확인
- **3초 규칙** 충족: 스크롤 없이 액션 가능

**시각적 처리:**
- 배경색 다르게 (약간 elevated)
- 마감일 색상 코딩 (🔴 D-0, 🟡 D-1~2, 🟢 D-3+)
- 인라인 액션 버튼 (완료, 진행중 등)

---

### 3. 인라인 액션 (Inline Actions)

**변경:** 카드/리스트 내에서 바로 액션 실행 가능

**이유:**
- 기존: 승인하려면 카드 클릭 → 상세 페이지 → 승인 버튼
- 개선: 대시보드에서 바로 [승인] [반려] 클릭
- **클릭 수 3회 → 1회로 감소**

**적용 위치:**
- 승인 대기 건: [승인] [반려] [상세]
- 긴급 작업: [완료] [진행중] [위임]
- 콘텐츠 발행: [발행] [수정] [취소]

---

### 4. 컨텍스트 힌트 (Contextual Hints)

**변경:** 인사말에 오늘 해야 할 일 요약 포함

**이유:**
- "좋은 아침이에요" 만으로는 정보 전달 안 됨
- "오늘 3건의 행사와 2건의 승인 요청" → 즉시 우선순위 파악
- 인지 부하 감소

**예시:**
```
총무팀: "오늘 3건의 행사와 2건의 승인 요청이 있습니다"
정책팀: "오늘 마감 문서 1건, 팩트체크 요청 2건이 있습니다"
홍보팀: "오늘 예약 발행 5건, 승인 대기 콘텐츠 2건이 있습니다"
```

---

### 5. 긴급도 시각화 (Urgency Visualization)

**변경:** D-Day 기반 색상 코딩 일관 적용

**이유:**
- 기존: 마감일이 텍스트로만 표시 → 긴급도 구분 어려움
- 개선: 색상으로 즉시 인지 가능

**색상 규칙:**
| D-Day | Color | Badge |
|-------|-------|-------|
| D-0 (오늘) | `rose-500` 🔴 | "오늘 마감" |
| D-1 ~ D-2 | `amber-500` 🟡 | "임박" |
| D-3 ~ D-7 | `emerald-500` 🟢 | "여유" |
| D-7+ | `slate-400` ⚪ | 표시 안 함 |
| 지남 | `rose-600` 🔴 | "지연" (bold) |

---

### 6. Quick Actions 바 (Quick Actions Bar)

**변경:** 대시보드 상단에 역할별 핵심 액션 3개 버튼

**이유:**
- 가장 자주 하는 행동을 1클릭으로 접근
- 네비게이션 메뉴 탐색 불필요
- **3초 규칙의 핵심**

**역할별 Quick Actions:**

| 역할 | Action 1 | Action 2 | Action 3 |
|------|----------|----------|----------|
| 총무팀 | 승인 처리 | 오늘 일정 | 요청 등록 |
| 정책팀 | 새 문서 | 팩트체크 | 오늘 브리핑 |
| 홍보팀 | 새 콘텐츠 | 발행 큐 | 분석 보기 |

## Implementation Priority

### Phase 1: Core Changes
1. Dashboard 개인화 로직 (역할 기반 섹션)
2. "내 작업" 섹션 추가
3. Quick Actions 바 구현
4. D-Day 색상 코딩

### Phase 2: Inline Actions
1. 승인 인라인 액션
2. 작업 상태 변경 인라인
3. Toast 피드백

### Phase 3: Polish
1. 인사말 컨텍스트 힌트
2. 역할별 빈 상태 메시지
3. 애니메이션 (카드 등장, 액션 피드백)

| Metric | Before | Target |
|--------|--------|--------|
| 첫 액션까지 시간 | ~15초 | <3초 |
